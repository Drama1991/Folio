## 设计决策

| 项 | 决定 | 理由 |
|---|---|---|
| 封面 | 居中 + 大尺寸 | 详情页该有"仪式感"，封面是主视觉 |
| 标题/作者/出版 | 居中 | 配合封面居中；与 Profile 一致风格 |
| 主 CTA | 全宽 `Mark` 按钮 + 4 个状态副 CTA segment | 一键直跳 RecordModal-sheet；或快速选状态不进表单 |
| 我的状态 | 仅 Mark 后出现 | 减少未 Mark 用户的视觉噪音 |
| 简介 | 默认折 4 行 + 展开 | 长简介占屏严重 |
| 长评 | 自己的在前，其他人在后 | "我的内容"高于"社区内容" |
| ⋯ 菜单 | 分享 / 复制链接 / 在 NeoDB 打开 / 举报 | 标准操作走 sheet |
| 整页滚 | 不固定标题区 | 屏寸优先，需要时再加 sticky 小标题 |
| FAB | ❌ 不显示 | Detail 页 Mark 已经是显式 CTA，FAB 反而冗余 |

## 副 CTA segment 行为

点 [想读] / [在看] / [已读] / [搁置] 任一：
- 已 Mark 状态 → 直接更新为新状态，toast 提示"已更新为：在看"
- 未 Mark → 调起 RecordModal-sheet，预选好对应状态

为什么这么设计：80% 的标记动作只需要"换个状态"，不需要进表单填评分/笔记。给一个"傻瓜路径"省 3 次点击。

## 路由与返回

- 从 Home 点入：← 返回 Home
- 从 Profile 最近条目点入：← 返回 Profile
- 从 Search 点入：← 返回 Search 结果列表（保留滚动位置）

## 与桌面端的差异

- ❌ 桌面是横向左侧封面 + 右侧元信息，移动端垂直堆叠
- ❌ 桌面隐藏 ⋯ 操作可能在右上 hover 显形 → 移动端 ⋯ 常显
- ❌ 移除 FAB（避免与主 CTA 重复）
- ✅ 保留所有内容板块：状态 / 简介 / 长评 / 其他人
